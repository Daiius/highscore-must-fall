# 05. 認証とプライバシー

> **状態: スケルトン**。本文は執筆予定。

## 章立て（予定）

- 認証: **better-auth**、ソーシャル OAuth のみ（**Google 初手**、GitHub/Discord 等を後で追加可能に）。メール+パスワードは持たない。
- owner_id = 認証済みユーザー ID（データ分離のセキュリティ境界、今すぐ有効）。
- 公開配置の前提: HTTPS / シークレット管理（`.env*` 非コミット）/ CORS /（将来）レート制限。
- プライバシー:
  - **MVP = private データのみ**。他ユーザーの個人データのピンポイント閲覧は不可。
  - ランキングは自分の run 内のみ。
  - 将来: 匿名の全体統計・横断ランキングに **オプトイン** でデータ利用。
  - データモデルに将来「集計利用同意フラグ（ユーザー単位）」を足す余地（MVP 未実装）。
  - 自己申告スコアの信頼性問題（横断ランキングは盛れる）→ 設計は将来。
- Phase2 の API/MCP はユーザー単位トークン（better-auth 発行）。
