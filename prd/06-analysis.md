# 06. 分析機能

> **状態: スケルトン**。本文は執筆予定。

## 章立て（予定）

### MVP（記述的）
- 基盤: run 一覧 / run 詳細（閲覧・編集・削除）/ 自己ベスト・自分の run 内ランキング。
- スコア推移: `final_score` × `played_at` の折れ線。
- アップグレード頻度: 全体 / 週別の取得回数ランキング。
- アップグレード取得タイミング分布: 特定 upgrade が何週・何手目で取られがちか。
- 最小カタログ管理: 未検証一覧 / verify / エイリアス統合（名寄せが崩れると記述分析が成立しないため MVP 前提）。

### Phase2（推論的・本命）
- アップグレード × タイミング → スコア相関（例: week1 で X を取った run の平均スコア vs 取らない run）。
- リロール回数 × スコア、リワード分析。
- ※ 数十〜数百 run 蓄積後に意味を持つ。MVP の集計・グラフ部品を再利用。
